DOMUS API ENDPOINTS

Flutter usage notes
- Use this as raw endpoint base.
- Use .agents/flutter_app_docs/00_general_flutter_plan.txt as architecture base.
- Then pass one screen TXT from .agents/flutter_app_docs per Flutter task.
- Parent and child/member screens should be separate pages/providers, not one giant role-conditional page.
- After POST actions, update UI with the returned entity/balance first, then refresh the list if needed.
- If an action returns unlocked_badges, show a badge earned modal/toast.

Auth
POST /api/register
Body: name, email, password, password_confirmation, terms_accepted
Returns: token, user

POST /api/login
Body: login, password
Returns: token, user

GET /api/me
Auth: bearer token
Returns: user, parent_admin_name, is_family_member

GET /api/user
Auth: bearer token
Returns: user

POST /api/logout
Auth: bearer token
Returns: message

GET /api/notifications
Auth: bearer token
Query optional: limit
Returns: notifications

Domus points
GET /api/domus-levels
Auth: bearer token
Returns: levels

GET /api/parent/domus-points
Auth: parent
Returns: members(user_id, name, role, historical_points, earned_points, spent_points, available_points, level), rewards, redemptions

POST /api/parent/domus-points/rewards
Auth: parent
Body: title, description, points_cost
Returns: message, reward

POST /api/parent/domus-points/redemptions/{redemption}/pay
Auth: parent
Returns: message, redemption

GET /api/child/domus-points
Auth: child/member
Returns: points, level, badges(id, slug, title, description, image_url, points_reward, is_completed), rewards, redemptions

POST /api/child/domus-points/rewards/{reward}/redeem
Auth: child/member
Returns: message, redemption, domus_points(points, level)

Goals
GET /api/child/goals
Auth: child/member
Returns: goals, balance_cents, balance_display

POST /api/child/goals
Auth: child/member
Body: name, description, target_amount optional
Returns: goal

POST /api/child/goals/{goal}/deposit
Auth: child/member
Body: amount
Returns: goal, balance_cents, balance_display

POST /api/child/goals/{goal}/withdraw
Auth: child/member
Body: amount
Returns: goal, balance_cents, balance_display

POST /api/child/goals/{goal}/complete
Auth: child/member
Returns: goal

POST /api/child/goals/{goal}/cancel
Auth: child/member
Returns: goal, balance_cents, balance_display

Balance
POST /api/balance/add
Auth: bearer token
Body: amount
Returns: balance_cents, balance_display, movement, unlocked_badges(id, slug, title, description, image_url, points_reward), domus_points

Transfers
GET /api/transfers
Auth: parent
Returns: transfers

POST /api/transfers
Auth: parent
Header: Idempotency-Key
Body: child_user_id, amount
Returns: transfer, remaining_parent_balance, remaining_parent_balance_display, child_balance_cents, child_balance_display

Cash withdrawals
GET /api/withdrawals
Auth: parent or child/member
Returns: withdrawals, requests, all

POST /api/parent/withdrawals
Auth: parent
Body: child_user_id, amount
Returns: withdrawal, child_balance(balance_cents, balance_display)

POST /api/child/withdrawals
Auth: child/member
Body: amount
Returns: withdrawal, child_balance(balance_cents, balance_display)

POST /api/withdrawals/{cashWithdrawal}/accept
Auth: pending side only
Returns: withdrawal, child_balance(balance_cents, balance_display)

POST /api/withdrawals/{cashWithdrawal}/cancel
Auth: parent or child/member in withdrawal
Returns: withdrawal, child_balance(balance_cents, balance_display)

Family members
GET /api/family-members
Auth: parent
Returns: members, loan_targets

GET /api/family-members-summary
Auth: parent
Returns: members

GET /api/family-members/{user}/summary
Auth: parent
Returns: member, loans

POST /api/family-members
Auth: parent
Body: name, username, password, password_confirmation, is_minor, guardian_declaration_accepted
Returns: member

Allowances
GET /api/allowances
Auth: parent
Returns: allowances, summary

POST /api/allowances
Auth: parent
Body: child_user_id, amount, frequency, first_payment_immediate, start_at
Returns: allowance, result, balances inside result when executed

POST /api/allowances/{allowance}/execute
Auth: parent
Returns: execution result, remaining_parent_balance_display, child_balance_display

Savings boxes

POST /api/savings-boxes

POST /api/savings-boxes/{savingsBox}/deposit

Body: amount
Returns: account, remaining_balance_cents, remaining_balance_display

POST /api/savings-boxes/{savingsBox}/withdraw
Auth: child/member

Loans
GET /api/loans
Auth: family user
Returns: loans

GET /api/loans/waiting
Auth: family user
Returns: loans

GET /api/loans/active-total
Auth: parent
Returns: total_active_loans, estimated_total_paid_cents, estimated_total_paid_display, interest_generated_this_month_cents, interest_generated_this_month_display, interest_generated_historical_cents, interest_generated_historical_display, currency

POST /api/loans
Auth: parent or child/member
Body: child_user_id, amount, reason, due_date, installments_count, installment_frequency, has_interest, interest_mode, annual_interest_rate, fixed_interest_amount
Returns: message, loan, remaining_balance and remaining_balance_display when parent creates an offer

POST /api/loans/{loan}/approve
Auth: parent
Returns: message, loan, remaining_balance, remaining_balance_display, child_balance_cents, child_balance_display

POST /api/loans/{loan}/respond
Auth: child/member
Body: action accept/reject, reason required when reject
Returns: message, loan, member_balance_cents, member_balance_display

POST /api/loan-payments/{loanPayment}/pay
Auth: child/member
Returns: message, payment, loan, member_balance_cents, member_balance_display, new_balance_cents, new_balance_display, parent_balance_cents, parent_balance_display, unlocked_badges(id, slug, title, description, image_url, points_reward), domus_points

Education

Auth: bearer token
Returns: categories

GET /api/education/categories/{category}/courses
Auth: bearer token
Returns: courses

GET /api/education/courses/{course}
Auth: bearer token
Returns: course

POST /api/education/lessons/{lesson}/complete
Auth: bearer token
Returns: message, lesson_completion, progress

POST /api/education/lesson-parts/{lessonPart}/submit

Body: answers[] with question_uuid and answer

Tasks
GET /api/parent/tasks
Auth: parent
Returns: tasks, available_balance_cents, available_balance_display

POST /api/parent/tasks
Auth: parent
Body: name, description, reward_amount, reward_points
Returns: task, available_balance_cents, available_balance_display

POST /api/parent/tasks/{task}/review
Auth: parent
Body: action approve/retry/cancel
Returns: message, task, available_balance_cents, available_balance_display when parent balance changed, available_child_balance_cents, available_child_balance_display when child balance changed, unlocked_badges(id, slug, title, description, image_url, points_reward), domus_points when points changed

GET /api/child/tasks
Auth: child/member
Returns: tasks

POST /api/child/tasks/{task}/accept
Auth: child/member
Returns: task

POST /api/tasks/member/completed/{task}
Auth: child/member
Returns: task, unlocked_badges
